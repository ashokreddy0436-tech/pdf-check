Q8. What is the Page Object Model (POM)?
Answer:
Page Object Model is a design pattern that enhances test maintenance and reduces code duplication by separating page-specific code (locators + actions) from test scripts.

Example:

public class LoginPage {
    WebDriver driver;
    @FindBy(id="username") WebElement username;
    @FindBy(id="password") WebElement password;
    @FindBy(id="login") WebElement loginBtn;

    public LoginPage(WebDriver driver) {
        PageFactory.initElements(driver, this);
    }

    public void login(String user, String pass) {
        username.sendKeys(user);
        password.sendKeys(pass);
        loginBtn.click();
    }
}

Q16. What are the types of waits in Selenium? When would you use each?
Answer:

Implicit Wait â€“ waits for a certain time before throwing exception.

Explicit Wait â€“ waits for a specific condition to occur.

Fluent Wait â€“ similar to explicit, but with polling frequency and exception handling.

Use Cases:

Use implicit for globally stable elements.

Use explicit for dynamic elements like popups, progress bars.

Use fluent when you need custom polling or handling exceptions like NoSuchElementException.

Q17. How do you handle StaleElementReferenceException?
Answer:
This happens when the DOM is updated and the element becomes obsolete.

Solutions:

Re-locate the element before interacting again.

Use WebDriverWait to wait for element to be visible or clickable.

Avoid storing WebElement as global/static variables for long.
